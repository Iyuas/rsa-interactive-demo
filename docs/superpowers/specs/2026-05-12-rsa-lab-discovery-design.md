# RSA Lab — Discovery-First Interactive Redesign

**Дата:** 2026-05-12
**Цель:** превратить три основных раздела (Key Generation, Encryption, Decryption) из формы "введи ответ + Show solution" в учебную лабораторию, где ученик без матбазы САМ открывает каждое понятие через интерактивные виджеты, потом видит формулу, потом проверяет себя.

## Ограничения

- Стиль/CSS/цвета не меняем. Используем существующие переменные `--t-*`.
- Математика остаётся в `src/utils/rsaMath.js`. Виджеты только потребляют трассы.
- Никаких новых npm-зависимостей. Анимации — `framer-motion` (уже стоит).
- `Theory.jsx`, `Layout.jsx`, `App.jsx`, `presets.js`, `ascii.js` — без изменений (или минимальные правки).
- Текст интерфейса — на русском (как в `Theory.jsx`).

## Педагогический паттерн: Discover → Reveal → Practice

Каждое нетривиальное понятие проходит три фазы:

1. **Discover** — интерактив без формулы. Виджет ставит вопрос ("что общего у этих чисел?"), даёт крутить/кликать/тащить. Ученик открывает закономерность сам.
2. **Reveal** — кнопка "Показать правило" раскрывает формулу и связывает её с тем, что ученик только что трогал. Под формулой — пошаговая визуализация на трассе из `rsaMath.js`.
3. **Practice** — `AnswerCheck` с трёхуровневыми подсказками: намёк ("вспомни эксперимент"), частичное решение, полное решение.

Compose-паттерн реализует общий компонент `Concept.jsx`.

## Принципы интуитивности (новое — добавлено по просьбе пользователя)

1. **Один экран — одна мысль.** Каждый подшаг занимает одну карточку. Прокрутки внутри карточки нет. Длинные объяснения сворачиваются в "Подробнее".
2. **Подписи везде.** У каждого числа на виджете маленькая подпись что это (`p = 61 → простое число`, `n = pq = 3233 → модуль`). Никаких немых формул.
3. **Кликабельный глоссарий.** Любой термин (`mod`, `gcd`, `φ(n)`, `обратный`, `простое`, `взаимно простые`) — кликабелен. Открывает поповер с микро-анимацией и одной фразой объяснения. Реализация в `Glossary.jsx`.
4. **Состояние всегда видимо.** Сверху каждого раздела — компактная "хлебная крошка" с уже найденными значениями (`p=61, q=53, n=3233, φ=3120, e=?, d=?`). Текущий шаг подсвечен.
5. **Прогресс по понятиям**, не по шагам. В сайдбаре под навигацией — список изучаемых концепций с галочками: "понимаю mod ✓", "понимаю gcd ✓", "понимаю простоту ✓". Цель — чтобы ученик видел "сколько ещё концепций впереди".
6. **Микро-анимации обратной связи.** Правильный ответ — короткий "пинг" зелёным с шкалой 1.04 (≤200ms). Неверный — лёгкое покачивание (3px, 180ms). Раскрытие шага — slide-down 220ms. Никаких длинных интро-анимаций.
7. **Никаких слепых "далее".** Кнопка "следующий шаг" появляется только когда понятие пройдено. До этого вместо кнопки — подсказка "Сначала разберись с эспериментом выше".
8. **Tooltip на иконках, не текст-простыня.** Все вспомогательные пояснения — в hover-tooltip с задержкой 150ms.
9. **Звукового сопровождения НЕТ** (web aesthetic).
10. **Mobile/responsive не приоритет** (демо рассчитано на desktop в классе).

## Распределение виджетов по разделам

### Key Generation

- Шаг 1 "Простые": `PrimeSieve` (Discover) → существующий `PresetPicker` остаётся (Practice).
- Шаг 2 "n = pq": быстрый Discover — два прямоугольника `p` и `q` физически сливаются в `n` через анимацию scale/translate. Reveal: формула. Practice: `AnswerCheck`.
- Шаг 3 "φ(n)": Discover — `GcdGrid` 1..n, ученик кликает числа взаимно простые с n; счётчик растёт. Когда счётчик = φ(n), всплывает "это и есть φ(n)". Reveal: формула `φ = (p-1)(q-1)`. Practice: `AnswerCheck`.
- Шаг 4 "Выбор e": `GcdGrid` сужается до 2..φ-1, подсвечивает gcd(e, φ)=1. Ученик выбирает кликом. Reveal: правило. Practice: `AnswerCheck` (как сейчас) + `EuclidTable`.
- Шаг 5 "Вычисление d": Discover — `EuclidStrip` (вперёд и обратная подстановка как анимированная лента). Reveal: правило с `BackSubList`. Practice: `AnswerCheck`.
- Шаг 6 "Сводка ключей": остаётся как есть, добавляем анимацию "сборки" ключа из найденных частей.

### Encryption

- Перед формулой `C = M^e mod n`: `ModClock` — Discover-виджет на маленьком модуле (например 7), ученик крутит стрелку и понимает, что значит "по модулю".
- Затем: `BitLadder` — Discover-виджет: показываем `M^e` как последовательность "square" и "multiply" побитово. Ученик прокликивает биты экспоненты.
- Существующий `AsciiTableViz` + посимвольное шифрование с `AnswerCheck` остаётся, но каждое поле обогащается подсказками с глоссарием.

### Decryption

- Сначала повтор `BitLadder` для `C^d mod n` (теперь ученик уже понимает).
- Новый Discover: `EulerOrbit` — кружочек, на нём точки 0..n-1. Выбираем M, виджет показывает прыжки `M → M·M → M^3 → ...`, через `ed` прыжков возвращаемся в M. Цель — интуиция теоремы Эйлера.
- Practice как сейчас.
- В конце — финальная анимация: ASCII-коды собираются обратно в буквы, буквы прилетают в строку plaintext.

## Анимации (бюджет)

- Все анимации ≤ 350ms, кроме "Reveal" пошаговой визуализации (auto-step 600ms между шагами).
- Используем `framer-motion`: `motion.div` с `layout`, `AnimatePresence` для появления/скрытия.
- Никаких `spring` с overshoot — для математики нужна точность.
- Reduced-motion: уважаем `prefers-reduced-motion: reduce` — отключаем все нефункциональные движения.

## Тестирование

- Существующие vitest-тесты на `rsaMath` и `ascii` остаются. Чисто визуальные виджеты не тестируем юнитами.

- Проверка `prefers-reduced-motion: reduce`.
